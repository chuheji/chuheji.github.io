element-ui关于轮播图问题的记录

发表于 2021-08-06 分类于 element 阅读次数： Valine：
本文字数： 9.3k 阅读时长 ≈ 8 分钟

背景

最近在做一些大屏的项目，其中一个模块是向上轮播,同时展示三个项目的形式，就使用element-plus的carousel去实现的

初步实现

为了同时展示三个项目需要使用overflow-y: visible;覆盖原本样式.并且外层容器要加上overflow: hidden;

本来以为这样就可以了，没想到element的动画和我想的有点不一样

源码

先去看看源码是怎么处理的

可以看到负责动画的类名是is-animating，它是由data.animate决定的，而每次切换时会触发translateItem方法，在translateItem方法中我们可以看到这样一行代码

原来他是只对了当前激活项目的前一个项目加了动画，后一个项目没有做处理。

修改源码肯定是不行的，所以我们尝试在自己的文件修改一下

最终实现

element的轮播图有一个change方法可以获取到当前激活项目的索引，所以我们需要做两步工作完成这个功能：

  1. 初始化时为索引0、1、2加上动画类名is-animating
  2. 当触发change方法时为activeIndex-1、activeIndex、activeIndex+1加上动画类名is-animating

react-weui重构主要问题汇总

发表于 2020-08-24 更新于 2020-09-30 分类于 typescript
本文字数： 4.7k 阅读时长 ≈ 4 分钟

Picker

  1. 不能将类型“(e: TouchEvent) => void”分配给类型“(event: TouchEvent) => void”。
    参数“e”和“event” 的类型不兼容。
    类型“TouchEvent”缺少类型“TouchEvent”的以下属性: which, cancelBubble, composed, returnValue 及其他 8 项。
    解决： handleTouchStart和handleTouchMove的参数e的类型设置为React.TouchEvent

  2. 类型“Element | Text”上不存在属性“clientWidth”。 类型“Text”上不存在属性“clientWidth”
    解决：将此类型断言为HTMLDivElement类型

  3. 如何在接口中声明数组的最大长度
    解决：没有直接限制的方式，只能使用元组，但是在此处由于数组不是固定长度不适合使用元组

  4. 元素隐式具有 “any” 类型，因为类型为 “string” 的表达式不能用于索引类型 “Data”。
    在类型 “Data” 上找不到具有类型为 “string” 的参数的索引签名
    解决：上面方法的数组参数类型设为Array

  5. [Intervention] Unable to preventDefault inside passive event listener due to target being treated as passive. See
    报错原因：google浏览器为了最快速的相应touch事件，做出的改变。
    解决：为需要preventDefault的容器加入css属性touch-action: none代替

  6. picker滑动时会触发下拉刷新
    解决：e.stopPropagation

  7. cityPicker报错，联动失效
    解决：cityPicker中updateGroup方法缺少参数，之前重构时由于此方法中看起来没有用到这几个参数，就给删除了，加入这几个参数并log出来就可以了，然后在这个方法中有一个picker.setState()方法导致报错，将其改为this.setState()即可

Msg

  1. 不能将类型“{ children: string | undefined; href: string | undefined; }”分配给类型“IntrinsicAttributes & FooterLinkProps”。
    类型“IntrinsicAttributes & FooterLinkProps”上不存在属性“href”
    解决：在FooterLink组件中加入href属性

  2. 导入声明与“Button”的局部声明冲突
    解决：接口名称Button与导入名称Button冲突，修改接口名称即可

Uploader

  1. 类型“Element | Text”上不存在属性“value”。
    类型“Element”上不存在属性“value”。uploader对象可能为 “null”
    解决: 将this.refs.uploader断言为HTMLInputElement类型

  2. 不能将类型“(_img: CanvasImageSource, sx: number, sy: number, sw: number, sh: number, dx: number, dy: number, dw: number, dh: number) => number”分配给类型“{ (image: CanvasImageSource, dx: number, dy: number): void; (image: CanvasImageSource, dx: number, dy: number, dw: number, dh: number): void; (image: CanvasImageSource, sx: number, sy: number, sw: number, sh: number, dx: number, dy: number, dw: number, dh: number): void; }”

  3. 如何将视频的预览图点击后播放视频
    解决：props中新添加一个方法currentVideo,参数为如果上传视频成功后视频的src，可以将此src存入state中，每次点击预览图后根据id索引播放对应视频

  4. 怎么将背景图片颜色变深
    解决：css的filter:brightness(70%)属性可以调整

  6. 上传图片和视频返回结果是没有数据的
    解决：文件上传类型是Blob类型，uploader组件将文件重新定义为自定义类型，其中包括一个nativeFile属性，它的值是最原始的文件，在上传时上传file.nativeFile即可（文件上传为空时，一定是文件类型或者是文件本身有问题）

Gallery

  1. gallery不支持视频，uploader需要点击预览视频
    解决：最初想要动态创建video标签，然后通过dom操作插入到gallery里面，这样虽然可以，但是复用性太差，考虑到这种类型的组件非常多，所以将gallery支持视频封装为基础组件，首先在gallery组件暴露一个属性isVideo，值为true时执行renderVideos方法渲染出视频

flex属性的作用

发表于 2024-01-25 分类于 css
本文字数： 1.6k 阅读时长 ≈ 1 分钟

前言

flex布局首次提出是2009年，2016年5月官方正式公布用于web端的最新稳定版本。平时使用flex布局也就是常用justify-content,align-items这两个属性。
而flex属性却是很少用，主要是不太理解用法，这次就详细了解一下flex属性吧

定义

flex属性是flex布局的三个属性的缩写：flex-grow flex-shrink flex-basis。关于这三个属性我的理解是：

  • flex-grow在父盒子还有多余空间时生效，规定多余空间子盒子各自所占有的份额。值越大占的份额越高
  • flex-shrink在父盒子空间不够使用溢出的情况下，各个子盒子需要缩放的份额。值越大需要缩放的越多，即占有份额越少
  • flex-basis即子盒子占有的固定份额

当flex只有一个非负数字值时，如：flex: 1，那1就代表flex-grow的值
当flex有两个非负数字值时，如：flex: 1 2，那就代表的是flex-grow: 1,flex-shrink: 2
当flex有一个表示长度的值时，如：flex: 200px,flex: 20%，那就代表的是flex-basis: 200px,flex-basis:20%

计算方式

优先级说明

计算尺寸的优先级为：最大最小尺寸限制 > 弹性增长或收缩 > 基础尺寸

所以max-width/max-height优先级最高，然后才是弹性盒子计算的尺寸，基础尺寸优先级最低

尺寸计算方式

  1. flex-grow

计算简单，根据比例分配即可

父盒子总宽度1000 - left子盒子基础宽度200 - right子盒子基础宽度200 = 600

600 / （left份额1 + center份额2 + right份额2） = 120

left宽度：120 * 1 + 200 = 320
center宽度：120 * 2 = 240
left宽度：120 * 2 + 200 = 440

  1. flex-shrink

这个计算麻烦一点，需要算出收缩的权重和

第一步：权重和计算。每个子盒的flex-shrinkflex-basis相加 1200+2300+3800 = 3200
第二步：超出宽度计算。 所有子盒子基本宽度-父盒子宽度 200+300+800-1000 = 300
第三部：计算需要收缩的宽度（flex-shrink * flex-basis / 权重和 * 超出宽度）

left宽度：基础宽度 - 收缩宽度 = 200 - (1 * 200 / 3200 * 300) = 181.25
center宽度：基础宽度 - 收缩宽度 = 300 - (2 * 300 / 3200 * 300) = 243.75
right宽度：基础宽度 - 收缩宽度 = 800 - (3 * 800 / 3200 * 300) = 575

看起来麻烦，但是我们一般使用的情况都是需要等比收缩等比放大的
